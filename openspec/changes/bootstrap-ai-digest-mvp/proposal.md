## Why

当前项目已经明确了后端内容链路方向，但产品层还没有形成统一规格，前端视觉目标、信息架构、内容展示方式与后端 API 边界也还没有被正式定义。继续直接写代码会导致三个问题：

1. 首页会停留在“资讯列表页”层面，难以做出高端、克制、有产品感的 AI 资讯站气质。
2. 前端可能过早依赖数据库结构，而不是围绕稳定的内容能力和 API 契约来设计。
3. 摘要、精选、日期分组、分类浏览这些核心体验容易在实现过程中被不断改口，造成返工。

现在需要先把 MVP 的产品骨架固定下来：做一个摘要优先、阅读舒适、适合技术人群的 AI 内容聚合首页，并明确它和内容 API、采集链路之间的关系。

## What Changes

本变更定义第一版 AI 资讯聚合平台的基础规格，覆盖：

- 网站外壳与响应式导航结构
- 首页内容流、日期分组、分类筛选与精选右栏
- 摘要优先的内容详情页
- 面向前端的公共内容 API 契约
- Phase 1 的内容采集、归一化与摘要生成链路

同时，本变更明确以下产品取舍：

- 品牌名暂时保留为 `caiji 的 AI 资讯`
- 首页保留右侧精选区域，以强化“编辑感”和高端产品气质
- 内容详情页优先展示摘要，而不是做成原文镜像页
- `Phase 1` 不做登录、收藏、全文搜索、推荐、后台运营台
- 前端组件基座统一使用 `shadcn/ui`

## Capabilities

### New Capabilities
- `site-shell`: 定义网站全局布局、导航、响应式结构、主题和 UI 组件基座
- `content-feed`: 定义首页内容流、日期分组、分类切换、卡片类型与精选右栏
- `summary-first-detail`: 定义摘要优先的内容详情体验和原文跳转方式
- `public-content-api`: 定义前端消费内容所需的公共 API 契约与边界
- `content-ingestion-pipeline`: 定义 Phase 1 的采集、归一化、摘要与内部任务能力

### Modified Capabilities
- None

## Impact

受影响的后续实现范围：

- Next.js App Router 页面结构与布局实现
- Tailwind CSS 设计 token 与主题变量
- `shadcn/ui` 组件初始化、引入与二次封装
- 首页、内容详情页、移动端抽屉导航等前端界面
- Node/Supabase 内容 API、摘要服务与采集任务
- Vercel 部署结构与环境变量管理
